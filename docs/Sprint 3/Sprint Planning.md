## 2. Manejo de tareas

### 2.1 Creación y Asignación

Para gestionar las tareas de manera eficiente, hemos utilizado ZenHub, como en los sprints anteriores. En este sprint, hemos continuado asignando revisores con antelación a la realización de las tareas, ya que esto mejoró nuestra eficiencia a la hora de realizar revisiones en el sprint anterior. 

Adicionalmente, tenemos en cuenta las dependencias y relaciones entre tareas. Por ejemplo, Borja realiza análisis sobre las APIs, pues él mismo ha implementado una.

Las tareas se crean en forma de issues de GitHub y cada una debe tener:

- Un nombre explicativo. (Ej.: Términos de servicio)
- Una etiqueta de identificación. (Ej.: A3.2.1)
- Una estimación de esfuerzo usando la estrategia planning poker para asignar los puntos de historia.
- Solo una persona asignada por tarea y otra persona (distinta) a su revisión. (Exceptuando los apartados A3.5 y A3.6)
- Una rama asociada.
- Las etiquetas que se consideren apropiadas.
- Milestone a la que pertenecen. (En este caso, "Sprint 3")

### 2.3 Seguimiento

Para el seguimiento de las tareas se utiliza el **dashboard de ZenHub** y las **Weekly Stand-Up Meetings**, que se realizan cada semana.

También se lleva un registro en **Clockify** de las horas que cada persona dedicó a cada tarea. Este nos proporciona dos documentos en formato pdf que nos permiten llevar a cabo un análisis final del rendimiento del sprint. El primero de ellos es un **informe detallado** con cada una de las tareas realizadas progresivamente por cada  miembro del grupo. El segundo es un **informe general** con gráficos más visuales. Ambos informes pueden encontrarse en la carpeta del _sprint 3_ que se encuentra dentro de la carpeta _docs_ del proyecto.

### 2.4. Estimaciones

Continuamos utilizando **Planning Poker** como técnica principal para estimar el esfuerzo de las tareas del Sprint, facilitado por ZenHub.

## 3. Conclusión

El éxito en la gestión de Sprints y tareas depende de una planificación clara, una ejecución disciplinada y una mejora continua basada en el aprendizaje del equipo. Aplicar estas prácticas permitirá mantener la eficiencia.
